This class provides APIs for registering and unregistering Actionscript-based event listeners for InCopy events.

The InCopy CSAW Library currently only allows the adding of event listeners written in JavaScript. This class enables listeners to be registered in pure Actionscript, by abstracting over the process of defining JavaScript listener functions.

This class should not be constructed directly. Instead, use the getInstance method to obtain a reference to the singleton ICScriptingEventAdapter instance.

Use the addEventListener and removeEventListener methods to add or remove event listeners to the root application object of the InCopy scripting DOM.

For example, to add a "new document" listener:

ICScriptingEventAdapter.getInstance().addEventListener(Event.AFTER_NEW, listener);

where listener has the following method signature:

function listener(event:com.adobe.incopy.Event):void

When a new document is opened, the listener function will be called, parameterised an Event object representing the event which triggered the call.

See com.adobe.incopy.Event and its subclasses in the InCopy CSAW Library for other allowable event types.

Use the addEventListenerToObject and removeEventListenerFromObject functions to add or remove event listeners to and from particular InCopy objects. For example, to add a "document closed" listener to a document:

var doc:Document = InCopy.app.activeDocument; ICScriptingEventAdapter.getInstance().addEventListenerToObject(DocumentEvent.AFTER_CLOSE, listener, doc);

Note that listeners registered through this class are called synchronously: InCopy will not continue to execute until all listeners for an event have completed execution.

addEventListener()method
override public function addEventListener(eventType:String, listener:Function, useCapture:Boolean = false, priority:int = 0, useWeakReference:Boolean = false):void

Adds a listener for the specified event to the application object at the root of the InCopy scripting DOM.

See com.adobe.incopy.Event and its subclasses for allowable event types.

If useCapture is set to true, the event may be handled by event listeners registered to scripting objects above the target object of the event during the capturing phase of event propagation. This means an event listener on the application, for example, can respond to a document event before an eventListener is triggered.

Parameters

eventType:String — The unique string identifier of the event to listen for
 
listener:Function — The listener function that processes the event
 
useCapture:Boolean (default = false) — True to allow the event to be handled in the capture phase of event propogation
 
priority:int (default = 0)
 
useWeakReference:Boolean (default = false)

addEventListenerToObject()method 
public function addEventListenerToObject(eventType:String, listener:Function, scriptingObj:CSHostObject, useCapture:Boolean = false):EventListener

Adds a listener for the specified event to the specified InCopy object. Returns an com.adobe.incopy.EventListener instance representing the added event listener.

See com.adobe.incopy.Event and its subclasses for allowable event types.

The scriptingObj parameter must be an InCopy CSAWLib class instance to which event listeners can be added. For example, an instance of com.adobe.incopy.Document. If scriptingObj is null, the event listener is added to the application object at the root of the InCopy scripting DOM.

If useCapture is set to true, the event may be handled by event listeners registered to scripting objects above the target object of the event during the capturing phase of event propagation. This means an event listener on the application, for example, can respond to a document event before an eventListener is triggered.

Parameters

eventType:String — The unique string identifier of the event to listen for
 
listener:Function — The listener function that processes the event
 
scriptingObj:CSHostObject — The InCopy object to add the event listener to
 
useCapture:Boolean (default = false) — True to allow the event to be handled in the capture phase of event propogation

Returns
EventListener — An EventListener representing the added event listener

removeEventListenerFromObject()method 
public function removeEventListenerFromObject(eventType:String, listener:Function, scriptingObj:CSHostObject, useCapture:Boolean = false):Boolean

Removes the specified listener for the specified event from the given InCopy object.

Returns true if the event listener was removed successfully, else returns false. Failure may occur if for example a listener is removed without ever being added.

If scriptingObj is null, the event listener is removed from the application object at the root of the InCopy scripting DOM.
